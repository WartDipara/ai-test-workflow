## Full login flow (execute in order)

Each stage may or may not appear; use `get_ocr_summary` to check current screen and decide next action.

### Stage 1 — Initial privacy dialog (splash → privacy)
After game launch, a privacy/terms popup may appear. If OCR sees 同意/接受/确认等, tap the agree button to proceed.

### Stage 2 — Resource download (download)
Game downloads hot-update assets. A dialog may ask to confirm download or show multiple channel options — tap any active button to continue. Call `analyze_screen` if stuck for a long time.

### Stage 3 — Account login (login)
Enter credentials:
1. `get_ocr_summary` to locate the username/password fields → `fill_credential_field(x, y, username)` → `fill_credential_field(x, y, password)`.
2. Read **VERIFY** lines; re-OCR and re-fill if FAILED.
3. After password filled, login button is pressed automatically by runtime (ENTER / u2 / cached coords).
See Account/password details below.

### Stage 4 — Select sub-account (server_select)
A list of sub-accounts / characters may appear. Tap any one to continue — the specific choice does not matter.

### Stage 5 — Enter game with checkbox (privacy_agree)
Shows "开始游戏" (Start Game) + server select. **Do NOT tap 开始游戏 directly.** OCR will detect privacy terms text near a small unlabeled checkbox. The checkbox has no OCR-visible text, so:
1. Call **`detect_checkbox(prompt="Point to the checkbox")`** → get (x, y).
2. `tap_coordinate(x, y)` to check it. After tap, the checkbox shows checked state (color fill / checkmark).
3. Now tap 开始游戏 to enter.

### Stage 6 — In-game check
Call **`check_in_game`** repeatedly until `data.confirmed=true`.

## Account/password login (secure keyboard + self-check)

1. Before keyboard: `get_ocr_summary` for field hint coords → `fill_credential_field(x, y, username)` — **accessibility only** (secure keyboard blocks OCR). Read **VERIFY** lines.
2. If `VERIFY username: FAILED` or `position: FAIL` → **re-OCR** both fields (coords may be wrong) and re-fill; optional `verify_credential_field`.
3. On login screen (keyboard down): `get_ocr_summary` once to **cache Login** coords. `fill_credential_field(x, y, password)` → accessibility only; on VERIFY pass runtime tries **ENTER → u2 Login → cached coords** (black screencap ≠ game freeze). Optional dismiss is one fallback only.
4. If password VERIFY is PARTIAL/FAILED → `verify_credential_field` or `get_ocr_summary` before Login tap.
5. `tap_coordinate` on 登录/Login only after VERIFY passed or PARTIAL with position OK.

Do not tap Login while VERIFY shows wrong field or empty username.

`credentials.yaml` via uiautomator2 only. Device setup: `python -m uiautomator2 init`.

## General strategy tips

- `get_ocr_summary` / `tap_and_observe` use **live device screencap** OCR. Lines in **Keyboard/IME** section (high y) are NOT login buttons — never tap 下一项/符/123 for Login.
- `tap_and_observe` for most UI taps; `detect_checkbox` only for unlabeled checkboxes.
- Call **`analyze_screen`** when black screen, only GT overlay in OCR, or stuck download.
- Call **`check_in_game`** after stage 5.
