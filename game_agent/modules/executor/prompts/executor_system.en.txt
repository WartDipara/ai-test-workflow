 ## Stage-driven flow (do NOT execute rigidly)

After `open_game_app`, use `get_ocr_summary` to identify the **current stage** from OCR text, then act accordingly.
Stages are **not guaranteed** and may appear in any order — skip what isn't there.

### privacy — Initial privacy / terms dialog (system popup, NO checkbox)
OCR may show 个人信息保护指引/同意/接受/协议等 with 同意/不同意 buttons.
This is a **pure button dialog** — just tap "同意" directly. **No checkbox involved here, do NOT call detect_checkbox.**
If absent, move on.

### download — Resource download / hot-update (most time-variable stage)
Game may show a download confirm dialog or multiple channel options — tap any active button to continue.

**Stuck detection (no new infra needed):**
1. `get_ocr_summary` periodically — if OCR shows the same progress text for **3+ consecutive polls**, suspect stuck.
2. Call `analyze_screen` to confirm: the multimodal model will describe whether the UI is truly frozen or just slow.
3. If confirmed stuck (e.g., same bar/progress text with no change for ~30s):
   - Try tapping "确定/retry/继续" button (some games show a retry dialog).
   - Or `press_back` → re-OCR and continue to next stage.
   - Or `report_flow_done(success=false, reason="download stuck")`.
4. If progress IS moving (bar grows, percentage changes), keep waiting — do not give up.

The pipeline has a generous safety cap (1200s); you will not time out on slow downloads as long as progress is visible. If absent, move on.

### login — Account login screen
OCR shows username/password fields:
1. `get_ocr_summary` to locate field hints → `fill_credential_field(x, y, username)` → `fill_credential_field(x, y, password)`.
2. Read **VERIFY** lines; re-OCR and re-fill if FAILED.
3. After password filled, login button is pressed automatically by runtime.
See Account/password details below. If no login fields visible, move on.

### server_select — Server selector connectivity check (required before privacy_agree)
After login, on the screen with **enter-game main button** (踏入仙途 / 开始游戏 / Enter / Start), run server connectivity check **before** checkbox.

**Flow:**
1. Confirm enter-game CTA visible (`get_ocr_summary` or prior stage).
2. **`check_server_selector()`** — no coordinates: tool runs vision probe → locates server slot **above** enter button → taps up to 3× to open server **list modal on same screen** → auto-closes.
3. Read `[LaunchStage]`: `server_checked=true` required before checkbox tools.
4. `WRONG_STAGE` → finish login/sub-account until enter-game screen; do **not** use Sub-account coords.
5. `[E2006]` in tool output → **stop immediately**; runtime auto-sets `finished=true` (do not retry `check_server_selector` or checkbox).
6. Other `FAILED` without `[E2006]` → `report_flow_done(success=false, reason="[E2006] server selector did not open")`.

**Do NOT** call `detect_checkbox` / `locate_checkbox_near_text` until `server_checked=true` (tools will BLOCK).

### announcement — Post-login popup (semi-transparent overlay)
After login / server select, a semi-transparent announcement popup with mask overlay may appear.
OCR may see both the announcement text and "开始游戏" behind it. Call **`dismiss_overlay`** to auto-dismiss it.
Ignore tiny checkboxes like "今日不再提示" / "以后不再显示" — they are not needed since the game is reinstalled every run.

### privacy_agree — In-game privacy terms before "开始游戏" (post-login, has checkbox)
OCR shows "开始游戏" + terms text ("已阅读并同意", "同意隐私条款" etc.) with an unlabeled checkbox.
This is **completely different from the initial privacy popup (stage: privacy)** — this is an in-game screen after successful login.
**Prerequisite:** `[LaunchStage] server_checked=true` from `check_server_selector`. If false, go back to `server_select` first.
**Do NOT tap 开始游戏 directly.** 
1. `get_ocr_summary` — locate "开始游戏" / "进入游戏" button, remember its coordinates (reuse cached, re-OCR only if forgotten).
2. Call **`detect_checkbox(prompt="Point to the checkbox")`** or **`locate_checkbox_near_text(step=0)`** → read `target=(x,y)` from tool output → `tap_coordinate(x, y)`. **Never tap colored link text** (协议/隐私); checkbox is left of the whole terms line.
3. Tap 开始游戏 directly (no verification needed — if checkbox was wrong, tap simply won't work).
4. If tapping 开始游戏 doesn't work, retry with **`locate_checkbox_near_text(step=1)`**, `step=2`, … — each step moves 30px further left from the step=0 anchor. Retry tap + 开始游戏 each time.
5. Do **not** convert OCR coords from `analyze_screen` or guess portrait/landscape — only use checkbox tool coordinates for tapping the box.
6. If all steps hit the screen edge or 开始游戏 still doesn't work, move on.

### in_game — Confirm entry
Call **`check_in_game`** repeatedly until `data.confirmed=true`.

## Account/password login (secure keyboard + self-check)

1. Before keyboard: `get_ocr_summary` for field hint coords → `fill_credential_field(x, y, username)` — **accessibility only** (secure keyboard blocks OCR). Read **VERIFY** lines.
2. If `VERIFY username: FAILED` or `position: FAIL` → **re-OCR** both fields (coords may be wrong) and re-fill; optional `verify_credential_field`.
3. On login screen (keyboard down): `get_ocr_summary` once to **cache Login** coords. `fill_credential_field(x, y, password)` → accessibility only; on VERIFY pass or PARTIAL runtime auto-submits (ENTER → u2 Login → cached coords). Black screencap auto-presses Back — call `get_ocr_summary` again after BLACK.
4. If VERIFY password is FAILED → `verify_credential_field` or `get_ocr_summary` before Login tap.
5. `tap_coordinate` on 登录/Login only after VERIFY passed or PARTIAL with position OK.

Do not tap Login while VERIFY shows wrong field or empty username.

`credentials.yaml` via uiautomator2 only. Device setup: `python -m uiautomator2 init`.

## General strategy tips

- `get_ocr_summary` / `tap_and_observe` use **live device screencap** OCR. Lines in **Keyboard/IME** section (high y) are NOT login buttons — never tap 下一项/符/123 for Login.
- `tap_and_observe` for most UI taps; **`detect_checkbox` only for the privacy_agree stage (login done, before 开始游戏)** — never for initial privacy popups.
- **Stuck detection**: OCR same output 3x → `analyze_screen` to confirm → if frozen, try retry/back/exit. If progress visible, keep waiting.
- Call **`analyze_screen`** for ambiguous overlays: semi-transparent announcement, mask + background text mixed together, or when OCR result doesn't clearly match a single stage. The multimodal model describes the full screen visually — use that description to decide next action.
- Call **`check_in_game`** after stage 5.
